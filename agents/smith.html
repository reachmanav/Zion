TECH STACK

PATTERN
Map-Reduce — split task into N pieces, run N workers in parallel, merge results
RUNTIME
Python asyncio.gather() — true parallel I/O across N LLM calls
ORCHESTRATOR
Trinity (Opus) on laptop OR Trinity-CLI on VM — either can spawn Smiths
WORKERS
Groq Llama 3.3 70B (fast, free tier) — perfect for sub-tasks. Falls back to Opus for complex pieces.
INVOKE
@trinity smith [task1; task2; task3] OR @trinity-cli smith [...]
MERGER
Trinity (Opus) — combines all Smith outputs into one clean reply
CONCURRENCY
Up to 8 parallel Smiths per task (Groq rate-limit safe)
FAILURE
If 1 Smith fails, Trinity reports partial result — no whole-task crash
COST
FREE — Groq free tier handles ~14,400 requests/day. Opus only for the merge step.
